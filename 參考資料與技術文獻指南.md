## 🔒 2. EKS 叢集與容器安全 (EKS & Container Security)

### 🛡️ EKS 安全最佳實踐 (面試/報告必引必考)
* **AWS EKS 安全最佳實踐指南**：[AWS EKS Best Practices Guide for Security](https://aws.github.io/aws-eks-best-practices/security/docs/)
  * *專題引用*：EKS 控制面 API Server 完全私有化（`EndpointPublicAccess: false`），徹底收斂公網暴露面。
* **EKS Access Entry 授權模式**：[EKS Access Entries Developer Guide](https://docs.aws.amazon.com/eks/latest/userguide/access-entries.html)
  * *專題引用*：捨棄舊版 `aws-auth` ConfigMap，全面改用最新的 EKS Access Entry API 對接 IAM Role。

### 臨時憑證授權與 Pod 識別
* **EKS Pod Identity 官方開發指南**：[Amazon EKS Pod Identity User Guide](https://docs.aws.amazon.com/eks/latest/userguide/pod-identities.html)
  * *專題引用*：捨棄複雜的 OIDC 信任與 IRSA，改用最新的 Pod Identity 派發臨時 Token 給網頁 Pod 與 K8sGPT Pod。
* **SSM Session Manager 跳板機管理**：[AWS Systems Manager Session Manager](https://docs.aws.amazon.com/systems-manager/latest/userguide/session-manager.html)
  * *專題引用*：Bastion Host 移除 Key Pair 與 Port 22 SSH 入口，改用 IAM 控制 SSM 隧道進行私有管理。

## 🗄️ 3. 資料保護與密鑰硬化 (Data Protection & Secret Hardening)

### 🔐 密碼不落地與動態解析
* **AWS Secrets Manager 動態讀取語法**：[AWS CloudFormation Dynamic References](https://docs.aws.amazon.com/AWSCloudFormation/latest/UserGuide/dynamic-references.html)
  * *專題引用*：在 RDS CloudFormation 中使用 `resolve:secretsmanager:SecretString` 自動生成與注入資料庫密碼，程式碼不含明文密碼。
* **RDS 儲存加密與安全備份**：[Encrypting Amazon RDS Resources](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Overview.Encryption.html)
  * *專題引用*：RDS MySQL 啟用預設 KMS 免費金鑰進行硬碟儲存加密（StorageEncrypted: true）與 DeletionPolicy: Delete 成本取捨。

## ⚙️ 5. 自動化修復與供應鏈安全 (Remediation & DevSecOps)

### 🦾 AWS CodeBuild 自動修復與 VPC 網絡
* **AWS CodeBuild 網絡接入最佳實踐**：[Use CodeBuild with Amazon Virtual Private Cloud](https://docs.aws.amazon.com/codebuild/latest/userguide/vpc-support.html)
  * *專題引用*：修復執行器 CodeBuild 置於 VPC 私有子網路，利用 EKS Node SG 繞過 EKS 控制面完全私有的存取限制。
* **防命令注入與防供應鏈劫持**：
  * **OWASP Command Injection Prevent Cheat Sheet**：[OWASP OS Command Injection Defense](https://cheatsheetseries.owasp.org/cheatsheets/OS_Command_Injection_Defense_Cheat_Sheet.html)
    * *專題引用*：Lambda 移除 `eval`，改為正規表達式白名單、結構化 Action JSON 與安全模板字串安全防護。
  * **kubectl 安全驗證**：[Kubernetes kubectl Installation Official Guide](https://kubernetes.io/docs/tasks/tools/install-kubectl-linux/)
    * *專題引用*：CodeBuild 自動抓取與叢集一致之 `v1.34.0` kubectl，並在下載時進行 `.sha256` 強制雜湊比對。
